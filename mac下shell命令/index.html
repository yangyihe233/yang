2.apt-get与brew

由于之前接触过Ubuntu的系统，所以比较熟悉一个便捷的安装软件包的命令sudo apt-get install xxx，一直觉得Mac也是Linux的系统和Ubuntu的区别应该不大，然而今天使用命令sudo apt-get install libxml2时Mac系统却报出了-bash:apt-get:command not found 这样的错误，后来上网查询才知道，原来apt-get是debian(Ubuntu)才有的包管理器，遂继续查询解决方案。

解决方案：使用brew代替apt-get

What is brew?

brew 全称Homebrew，是Mac OSX上的软件包管理工具，能在Mac中方便的安装软件或者卸载软件。

1.cat命令

用于查看文件内容，结合|grep可以对单行文本进行过滤（即显示出含有过滤字符串的当前行）

例子：cat test.txt |grep handsome

注意：linux过滤用grep，windows则用findstr

2.echo/touch/open

用于创建文件和写文件

用法：

1）touch test.txt 创建test.txt文件

2) open text.txt打开text.txt文件

3）cat content > text.txt 创建text.txt文件并写入content

4）cat list >> text.txt 向text.txt追加内容list

3.adb shell

1）adb shell dumpsys activity top 查看当前应用activity信息

2）adb shell dumpsys package com.tencent.mm 查看指定包名应用的信息

3）adb shell dumpsys meminfo [pname/pid] 查看指定进程名或进程ID对应的内存信息

4）adb shell dumpsys dbinfo com.tencent.mm 查看指定包名的数据库存贮信息

5）adb shell screencap -p /sdcard/t.png 截屏保存到设备

6）adb shell screenrecord –time-limit 5 /sdcard/demo.mp4 录屏保存到设备

7）adb shell input text 123456 输入文本到输入框（密码，长文本用起来方便）

8) adb install demo.apk 安装demo.apk,如果已经安装则用adb install -r demo.apk

9) adb uninstall demo.apk 卸载demo.apk

10) adb pull /sdcard/text.txt ~/ 把设备上的文件放到本地，可能会有权限问题需用chmod修改权限

11）adb push ~/text.txt /sdcard/ 把本地的文件放到设备上面，可能会有权限问题需用chmod修改权限

12）adb forward 【远端协议：端口号】【设备协议：端口号】

13）adb jdwp 查看设备中可以被调试的进程号

14) adb logcat -s tag 查看logcat日志

4.shell

1) am start -D -n com.android.browser/com.android.browser.BrowserActivity 用Debug方式启动应用

2）am startservice -n com.android.test/com.android.test.CusService 启动服务

3）am broadcast -a com.android.intent.ACTION.LOGIN 发送广播

4）netcfg 查看设备ip

5）aapt dump xmltree /data/demo.apk activity_main.xml 查看xml资源内容

6）dexdump /data/test.dex 查看dex文件的详细信息

7）cat /proc/[pid]/maps 查看当前进程的内存映射信息，比如加载了那些so和dex等

8）cat /proc/[pid]/status 查看当前进程的状态信息
